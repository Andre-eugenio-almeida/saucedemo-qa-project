# Plano de Testes

## 1. Informações do Projeto

**Projeto:** SauceDemo QA Project

**Versão:** 1.0

**Responsável:** André Almeida

**Data:** Julho/2026

---

# 2. Objetivo

Planejar as atividades de teste para validar as principais funcionalidades do sistema SauceDemo, garantindo que os requisitos funcionais sejam atendidos antes da entrega.

---

# 3. Escopo

### Funcionalidades que serão testadas

* Login
* Logout
* Listagem de Produtos
* Carrinho de Compras
* Checkout
* Menu Lateral
* Ordenação de Produtos

---

# 4. Fora do Escopo

Não fazem parte deste projeto:

* Integrações externas
* Gateway de pagamento
* Testes de segurança avançados
* Testes de carga
* Testes de acessibilidade

---

# 5. Tipos de Testes

## Testes Funcionais

Verificar se cada funcionalidade atende aos requisitos definidos.

## Testes Exploratórios

Explorar o sistema para identificar comportamentos inesperados.

## Testes de Regressão

Garantir que alterações não impactem funcionalidades existentes.

## Testes de Interface

Validar elementos visuais e fluxo de navegação.

## Testes de API

Validar os serviços REST utilizados pela aplicação.

## Testes Automatizados

Automatizar cenários críticos utilizando Robot Framework e Cypress.

---

# 6. Critérios de Entrada

Os testes poderão iniciar quando:

* Ambiente disponível.
* Aplicação funcionando.
* Requisitos definidos.
* Casos de teste criados.

---

# 7. Critérios de Saída

Os testes serão considerados concluídos quando:

* Todos os casos de teste forem executados.
* Bugs críticos forem resolvidos.
* Relatório de execução for concluído.
